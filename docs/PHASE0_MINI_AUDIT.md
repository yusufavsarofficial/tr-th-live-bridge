# PHASE 0 Mini Audit

## 1. Project type
- Mixed project, not yet a clean Android social app.
- Web/PWA: `public/index.html`, `public/app.js`, `public/sw.js`, `public/manifest.webmanifest`.
- Electron desktop: `desktop/main.js`, `package.json` has `desktop` and Windows packaging scripts.
- Backend: root `server.js` is CommonJS Node/Express/Socket.IO; `_render_repo/backend/` is separate TypeScript backend.
- Android: no root `android/`; `_render_repo/android/` contains a Kotlin Android project/prototype.
- Render/deploy: no root `render.yaml`; `_render_repo/render.yaml` exists.
- APK builder: root `scripts/build-apk.js` generates a WebView APK wrapper.

## 2. Main blockers
- 2-person limit:
- Root app is named/described as private 2-person chat in `package.json` and `README.md`.
- `server.js` defaults `MAX_USERS` to `2`, uses one `pingle-room`, and rejects the third user.
- Tests in `scripts/smoke-test.js` expect the third user to fail.
- `_render_repo/backend/src/socket/rooms.ts` defaults `maxParticipants` to `2`.
- `_render_repo/backend/src/socket/signaling.ts` creates `RoomStore(2)`.
- Android/backend Socket.IO mismatch:
- Root backend/web uses `join`, `message`, `users`, `typing`, `call:offer`, `call:answer`, `call:ice-candidate`, `call:end`.
- `_render_repo/android/.../MainActivity.kt` emits `join-room`, `message:send`, `call:start`, `call:end`.
- Android listens for `room-state`, `message:received`, `call:incoming`, `call:state`, `call:ended`.
- `_render_repo/backend` accepts `join-room` and call relay events, but does not handle `message:send`.
- `_render_repo/backend` emits `room:participant-joined`/`room:participant-left`, not Android `room-state`.
- REST endpoint mismatch:
- Android calls `/api/translate`.
- `_render_repo/backend/src/server.ts` mounts translation at `/translate`.
- Root `server.js` has no `/api/translate`; translation is internal to `message`.
- Fake APK/WebView wrapper problem:
- `npm run pack:apk` uses `scripts/build-apk.js`, which generates Java `MainActivity` with `android.webkit.WebView`.
- That APK is a website wrapper, not the target native Kotlin app.
- `_render_repo/android` has Kotlin UI/socket code, but it is not wired as a production app yet.
- Missing real WebRTC:
- Root web has browser WebRTC signaling, still scoped to 2-person room behavior.
- `_render_repo/android` has call UI/events but no native WebRTC media stack/SDP flow.
- Missing real STT/TTS:
- `_render_repo/backend/src/translation/speechToText.ts` returns 501.
- `_render_repo/backend/src/translation/textToSpeech.ts` returns 501.
- Security files that should not be committed:
- Found: `.env`, `node_modules/`, `dist/`, `.apk-keystore/`.
- Examples only: `.env.example`, `_render_repo/.env.example`, `_render_repo/backend/.env.example`.
- Generated by APK script if run: `.apk-build/`, `dist/Pingle.apk`, `.apk-keystore/password.txt`, `*.jks`.
- Root `.git` folder was not present in this workspace.

## 3. Keep / remove / refactor
- KEEP:
- `public/assets/pingle-logo.svg` and other user assets.
- `_render_repo/backend/` TypeScript backend pieces as reference code.
- `_render_repo/android/` Kotlin code as a native prototype/reference.
- Existing root web/Electron demo until replacement is planned.
- REMOVE LATER:
- WebView APK builder path: `scripts/build-apk.js`, `.apk-build/`, generated APKs.
- Duplicate/nested `_render_repo/` layout after code is migrated into root `backend/` and `android/`.
- 2-person-only smoke assumptions after multi-user architecture is defined.
- REFACTOR:
- Move target backend to root `backend/`.
- Move target Android app to root `android/`.
- Define one shared REST and Socket.IO contract before feature work.
- Replace in-memory single-room state with Supabase-backed users/conversations/messages.

## 4. Required target architecture
- `android/` = native Kotlin Android app.
- `backend/` = Node.js TypeScript API + Socket.IO.
- `database` = Supabase PostgreSQL.
- `calls` = WebRTC.
- `realtime` = Socket.IO.
- `translation/STT/TTS` = backend services.
